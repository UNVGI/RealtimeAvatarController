## 2. 名前空間確認

ルート名前空間 `RealtimeAvatarController.MoCap.VMC` を基点とし、
Internal/Editor サブディレクトリにはサブ名前空間を採用。
design.md §14 は名前空間階層を明示していないため、C# 慣例 (ディレクトリ階層 = 名前空間階層) に整合する設計とした。
Internal クラスは `internal` アクセス修飾子で公開を制限し、tasks.md 12-2 の
「Internal クラスは同名前空間の `internal`」要件 (= ルート名前空間配下にあり、
public ではなく internal で公開) を満たす。

| ファイル                                  | 宣言名前空間                                        | アクセス修飾子      | 結果 |
|-------------------------------------------|-----------------------------------------------------|--------------------|------|
| VmcMoCapSource.cs                         | RealtimeAvatarController.MoCap.VMC                  | public sealed      | OK   |
| VMCMoCapSourceConfig.cs                   | RealtimeAvatarController.MoCap.VMC                  | public             | OK   |
| VMCMoCapSourceFactory.cs                  | RealtimeAvatarController.MoCap.VMC                  | public sealed      | OK   |
| VmcFrameBuilder.cs                        | RealtimeAvatarController.MoCap.VMC.Internal         | internal sealed    | OK   |
| VmcMessageRouter.cs                       | RealtimeAvatarController.MoCap.VMC.Internal         | internal sealed    | OK   |
| VmcBoneMapper.cs                          | RealtimeAvatarController.MoCap.VMC.Internal         | internal static    | OK   |
| VmcOscAdapter.cs                          | RealtimeAvatarController.MoCap.VMC.Internal         | internal sealed    | OK   |
| VmcMoCapSourceFactoryEditorRegistrar.cs   | RealtimeAvatarController.MoCap.VMC.Editor           | internal static    | OK   |

すべて整合。
