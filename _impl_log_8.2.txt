8.2 RegistryLocator の EditMode テストを TDD で実装する

## 実装内容

Tests/EditMode/slot-core/RegistryLocatorTests.cs を新規作成し、
RegistryLocator の以下のテスト観点を NUnit の EditMode テストで検証する。

### テスト観点 (Requirements 11.3, 11.4, 11.5, 14.3)

1. 遅延初期化 (Req 11.5)
   - 各プロパティ (ProviderRegistry / MoCapSourceRegistry / FacialControllerRegistry /
     LipSyncSourceRegistry / ErrorChannel) の最初のアクセスで Default* 具象インスタンスが生成される
   - 同一プロパティへの複数回アクセスで同一インスタンスが返る (Interlocked.CompareExchange の動作確認)

2. ResetForTest の挙動 (Req 11.4)
   - 各 Registry / ErrorChannel が ResetForTest 後の再アクセスで新インスタンスになる
   - s_suppressedErrors HashSet が Clear される

3. Override*() の挙動 (Req 11.3)
   - 5 本の Override*() すべてがモック差し替えを反映する
   - 既定インスタンス生成後に Override で上書き可能
   - Override 後に ResetForTest を呼ぶと次回アクセスで既定実装に復帰する

4. [SetUp] / [TearDown] で RegistryLocator.ResetForTest() を呼ぶ (他テストとの状態共有を防止)

### スタブ実装

Override テスト向けに以下のインターフェース最小実装を同ファイル内に private 定義:
- StubProviderRegistry / StubMoCapSourceRegistry / StubFacialControllerRegistry /
  StubLipSyncSourceRegistry / StubSlotErrorChannel (UniRx Subject を使用)

### 補足

- DefaultProviderRegistry 等は internal sealed だが AssemblyInfo.cs の
  InternalsVisibleTo("RealtimeAvatarController.Core.Tests.EditMode") により参照可能。
- RegistryLocator.s_suppressedErrors も internal のため直接検証可能。
- 既存の RegistryLocatorResetTests.cs (d6db0f6 で追加された最小サニティテスト) は
  RegistryLocatorTests.cs に内容が包含されるため残置しても重複実行のみで機能影響なし。
