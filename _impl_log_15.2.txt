Task 15.2: SlotRegistry の internal スコープを徹底する (validation-design.md [N-3] 対応)

## 実施内容

### コードレビュー確認
- `Runtime/Core/Slot/SlotRegistry.cs`: `internal sealed class SlotRegistry` として定義済み (タスク 11.1 にて実装)
- `Runtime/Core/AssemblyInfo.cs`: `[assembly: InternalsVisibleTo("RealtimeAvatarController.Core.Tests.EditMode")]` 設定済み (タスク 8.3 にて対応)
- `SlotManager.cs` の `_slotRegistry` フィールドは `private readonly` で外部非公開
- パッケージ外へ SlotRegistry を公開する API は存在しないことを grep で確認

### テストによる invariant 固定
`Tests/EditMode/slot-core/SlotRegistryTests.cs` にリフレクションベースのテストを 3 件追加:
- `SlotRegistry_Type_IsNotPublic`: `type.IsNotPublic == true` / `type.IsPublic == false` を検証
- `SlotRegistry_Type_IsSealed`: `type.IsSealed == true` を検証
- `SlotRegistry_Type_IsVisibleToTestsViaInternalsVisibleTo`: InternalsVisibleTo 経由でテストからアクセス可能であることを確認

これにより、将来 `public` へ昇格したり `sealed` が外されたりした場合はテストが失敗して気付ける状態になった。

## 要件対応
- Requirements: 2.1, 2.2
- validation-design.md [N-3] 完了
