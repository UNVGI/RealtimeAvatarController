### 2. 既存実装の確認 (T2-3 で実装済み)
ファイル: `Packages/com.cysharp.realtimeavatarcontroller/Samples~/UI/Editor/SlotSettingsEditor.cs`
`RefreshTypeIds()` (L35-L52) に T7-3 で指定された try-catch 実装が既に存在する:

```csharp
private void RefreshTypeIds()
{
    try
    {
        _providerTypeIds    = RegistryLocator.ProviderRegistry
                                             .GetRegisteredTypeIds()
                                             .ToArray();
        _moCapSourceTypeIds = RegistryLocator.MoCapSourceRegistry
                                             .GetRegisteredTypeIds()
                                             .ToArray();
    }
    catch (System.Exception ex)
    {
        Debug.LogWarning($"[SlotSettingsEditor] Registry 候補取得失敗: {ex.Message}");
        _providerTypeIds    = System.Array.Empty<string>();
        _moCapSourceTypeIds = System.Array.Empty<string>();
    }
}
```

検証項目:
- [x] try ブロック内で `RegistryLocator.ProviderRegistry.GetRegisteredTypeIds()` と
      `RegistryLocator.MoCapSourceRegistry.GetRegisteredTypeIds()` を呼び出している
- [x] `catch (System.Exception ex)` により `NullReferenceException` を含む
      全ての `System.Exception` 派生例外を捕捉する
- [x] 例外捕捉時は `Debug.LogWarning` で警告を出力 (Editor コンソールに通知)
- [x] 例外時は `_providerTypeIds` / `_moCapSourceTypeIds` を
      `System.Array.Empty<string>()` にフォールバックする

### 3. null Registry 経路の網羅性
`RegistryLocator` の公開プロパティ
(`Runtime/Core/Locator/RegistryLocator.cs` L70-L79) は
`Interlocked.CompareExchange` による遅延初期化で通常は null を返さない。
ただし以下のケースでは `catch` が最後の防波堤として機能する:

- テスト経由で `OverrideProviderRegistry(null)` が呼ばれた直後に
  プロパティアクセス前の再 null 代入等で null が漏れた場合
- `GetRegisteredTypeIds()` 実装内部での `NullReferenceException` /
  `InvalidOperationException` / その他ランタイム例外
- `.ToArray()` 等の LINQ 拡張メソッドが null シーケンスに対して投げる `ArgumentNullException`

`System.Exception` はこれら全てのスーパークラスであるため、
`catch (System.Exception ex)` 句で完全捕捉される。

### 4. MessageType.Warning 表示への連携
例外捕捉時に `_providerTypeIds` / `_moCapSourceTypeIds` が空配列化されることで、
`DrawAvatarProviderSection()` (L88 `_providerTypeIds.Length == 0`) および
`DrawMoCapSourceSection()` (L120 `options == null`) の分岐が成立し、
以下の `MessageType.Warning` HelpBox が表示される:

- Provider: "Registry に Provider が未登録です。\n[InitializeOnLoadMethod] が実行されているか確認してください。"
- MoCap:    "Registry に MoCapSource が未登録です。\n[InitializeOnLoadMethod] が実行されているか確認してください。"

つまり null Registry でも UI は HelpBox + 手入力フィールドのフォールバックに
合流するため、ユーザーには例外ではなく警告として提示される。
